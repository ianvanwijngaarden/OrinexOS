# Orinex OS

Orinex OS is a custom desktop operating system designed with a focus on performance, structure, and simplicity. It is built from the ground up with a structured kernel architecture and an early native runtime, allowing applications to run natively without relying on heavy compatibility layers. The project is intended as both a practical system and a demonstration of operating system design, with an emphasis on clarity, structure, efficiency, and maintainability.


## Overview

Orinex OS aims to provide a clean and understandable platform while remaining capable enough to support real applications. The system is designed to run on modest hardware and avoids unnecessary complexity wherever possible. Key goals include:

* A structured and maintainable kernel design
* Early integration of a native userspace runtime
* Support for native C/C++ applications
* A lightweight application model with optional sandboxing
* Low resource usage suitable for entry-level systems
* A coherent and predictable system architecture


## Architecture

Orinex OS uses a structured monolithic kernel. Core system functionality such as memory management, scheduling, and device interaction is handled within the kernel, while higher-level services and applications run in userspace. The system is built around a defined userspace ABI, allowing applications to be developed and executed consistently without special handling. The application model supports:

* Native binaries compiled specifically for Orinex OS
* A lightweight runtime layer for simplified or sandboxed applications


## Current Status

Orinex OS is under active development. The project is focused on establishing a stable foundation before expanding into more advanced features. Current areas of development include:

* Core kernel functionality
* Executable loading and process management
* Userspace runtime and standard library
* Basic system utilities and shell environment
* Graphics subsystem and windowing
* Toolchain support for native development


## Roadmap

Planned milestones:

1. Stable userspace ABI and executable format
2. Functional libc/runtime environment
3. Cross-compilation toolchain
4. Basic desktop environment and window system
5. Native application support (editor, file manager, terminal)
6. Lightweight web viewer
7. Native compiler support
